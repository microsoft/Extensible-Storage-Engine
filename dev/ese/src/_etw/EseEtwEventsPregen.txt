//	Instructions for adding new Reguar ETW Event Traces:
//
//		1. Add a trace entry to bottom of START_TRACE_LIST
//			A) BaseTraceName - Ideally it should have both an ESE sub-component in front, and basic and short name.
//			B) Pick the next event number in order for this 2nd field.
//			C) Event verbosity level from ETW schema, such as: win:Information, win:Error, etc.
//			D) Quoted list of keywords (must match a keyword sepcified at top of Microsoft-ETW-ESE.mc file).
//
//			Note: Traces ending in _Start or _Stop are treated specially to mark the event as opcode:"win:Start/Stop".
//
//		2. Add a definition at end of START_TRACE_DEFNS (should be matched / in-order with entry for step 1).
//			A) Line that is "Define: BaseTraceName" to indicate start of trace field rows.
//			B) A list of fields / arguments for the trace, which are merged into the .mc file.
//				Note: If you add a new inType= that the gengenetw.pl script doesn't undersand, you may have
//				to modify the gengenetw.pl perl script as well.
//
//			IMPORTANT NOTE: Keep list #1 and #2 in same order.
//
//		3. Run gengen.bat
//
//		4. Add a call in target code to the function generated into EseEventTrace.g.hxx, it will be the BaseTraceName
//			you used for 1.A, prepended with "TR".  So CacheFirstDirtyPage becomes TRCacheFirstDirtyPage().
